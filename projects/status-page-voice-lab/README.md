# Status Page Voice Lab

Compose live status-page updates under uncertainty and watch how wording changes trust, clarity, ticket load, and operator calm.

## Core Idea

Most incident communication tools treat the message as obvious once the facts are known. This project treats public incident writing as a real systems decision: the team has partial truth, outside pressure, and a limited chance to shape behavior without overpromising.

## Why It Clears The Portfolio Bar

- The player is not tweaking parameters. They are authoring public language under pressure.
- The payoff is consequential: the same incident facts land differently depending on tone, certainty, timing promises, and user action guidance.
- The main lesson comes from consequence, not annotation.
- It is a different mechanic from the existing decision labs because the verb is composition, not budget allocation.

## Features

- Four incident beats with different truth windows and stakeholder pressure.
- Clause-based update composer across tone, certainty, timing promise, and action ask.
- Message preview before publish.
- Outcome scoring across trust, clarity, ticket load, and operator calm.
- Round-by-round verdict log plus end-of-session posture summary.
- Copyable decision brief for portfolio walkthroughs.
