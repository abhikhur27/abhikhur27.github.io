# Return Window Tribunal

Return Window Tribunal is a static decision game about refund policy under launch pressure.

## Core Idea

You rule on a queue of messy return cases where the "correct" answer is not purely legal or purely financial. Every decision changes four shared pressures:

- customer trust
- abuse pressure
- support load
- margin discipline

The point is not just to optimize one metric. It is to feel how repeated exceptions, strict denials, and compromise rulings become policy precedent for the rest of the queue.

## Why It Clears The Portfolio Bar

- The user has a concrete role: return-policy lead during a launch.
- Different rulings create different policy identities, not just different chart shapes.
- The payoff is experiential: each case feels different because the queue remembers what you already normalized.
- The interface is a judgment loop, not a parameter sweeper.
